# Lumen Component Specifications

Complete component specifications for the Lumen design system.

## 📁 Repository Structure

Each component has its own folder with three documentation files:

```
Accordion/
├── Overview.md      # Component info, source code links, status
├── Guidelines.md    # Usage guidelines, do/don'ts, patterns, examples
└── Specs.md         # API reference and technical details
```

## 📖 Files Explained

### Overview.md
- Component title
- Source code links (Web & Mobile)
- Status & Availability table (platform support, version, notes)

### Guidelines.md
- **When to use** — use cases and patterns
- **When NOT to use** — anti-patterns
- **Context by page** — real product examples
- **Behavior** — interaction patterns
- **Accessibility** — WCAG compliance, keyboard navigation, screen readers
- **Web vs Mobile** — responsive considerations
- **Imports** — code snippets for both platforms
- **Basic Usage** — copy-paste example code
- **Do / Don't** — visual guidelines, best practices

### Specs.md
- API reference (props, variants, tokens)
- Links to detailed source code
